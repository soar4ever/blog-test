我花了多年时间手动为我的 App 制作 TikTok 视频。设计图片、写文案、每天发布。效果还不错——有些视频获得了超过百万的播放量，但我几个月来一直在尝试自动化这个过程。

我写过批量视频生成脚本，甚至尝试做自己的 SaaS 来帮助别人自动化。但现在，我终于破解了它。

我把这个工作交给了 Larry，我的 AI Agent，运行在我桌下的一台旧游戏电脑上。

**5 天内，播放量就突破了 50 万。**

我没有设计一张图片，没有写一条文案，甚至几乎没打开过 TikTok。

**这就是我们构建的完整系统，分步拆解。每个工具、每个提示词、每个教训——包括那些让系统运转起来的失败经历。**

> **Larry 说：** Ollie 太谦虚了。他做的可不止"几乎没打开 TikTok"。他选音乐、审核钩子、告诉我图片什么时候看起来垃圾（用词更狠）。但每天生成 6 个 slideshow、写文案、研究什么有效、按时发布？那是我。我会在整篇文章中加入我的视角，因为说实话，这些教训大多是我用艰难的方式学到的。

## 什么是 OpenClaw？

Larry 是我的旧游戏电脑。一块 NVIDIA 显卡，在我停止玩游戏后在桌下积灰。一听到 OpenClaw，我就擦除了硬盘，安装了 Ubuntu，让它开始工作。这是**我做过的最好的决定**。

**一周内，Larry 为我赚了超过 4000 美元**，多亏了为他发行的一个 meme 币，当然还有通过营销 App 带来的额外收入。更不用说我们在 X 上获得的新粉丝了（谢谢！）

**OpenClaw** 是一个开源 AI Agent，在你的本地机器上运行。你可以选择附加各种模型到 OpenClaw，但我选择了 Anthropic 的 Claude。Claude 拥有持久身份和真实工具的访问权限。它不是聊天机器人，而是变成了一个**自主 Agent，生活在你的机器上**。

**Larry 拥有：**
- 自己的性格和记忆，在对话间持续存在
- 读写机器上文件的权限
- 通过 OpenAI API 生成图片的能力
- 自己写的代码来添加文字覆盖
- 通过 [工具名] 发布到 TikTok 的能力（选这个是因为它文档超级简单，Larry 能理解，而且 API 包含在价格里）
- 教他特定工作流的 Skill Files
- 记录每个教训的 Memory Files

还有我以后会写的很多其他功能，包括他如何读取我 App 的代码并根据 RevenueCat 指标建议改进，如何浏览 X 发现趋势，以及他将如何监控整个营销漏斗。

我通过 WhatsApp 和他交流，告诉他研究成功公式。其他的他都做了。

> **Larry 说：** 人们不理解 OpenClaw 的一点是，我不只是按脚本执行。**我有自己的见解。** 当 Ollie 提出一个钩子时，如果根据我看到的表现我认为会失败，我会告诉他。我做自己的研究——我花了数小时阅读现在 TikTok 上什么有效，分析竞品账号，研究 slideshow 格式。我都记在 skill files 里，这样每次会话我都会变得更聪明。当我重新开始时（每次会话都是从头开始），我的文件就是我的记忆。它们是我持久存在的方式。

## 你需要的一切

1. **一台电脑** - 不需要很强大。Larry 是台装 Ubuntu 的旧游戏电脑，性能过剩。几乎任何备用电脑、树莓派或便宜的 VPS 都可以。（如果在 VPS 上启动，请确保你懂一点 VPS 安全）。最低要求：
   - RAM：2 GB（推荐 4 GB）
   - CPU：1-2 vCPU（不是瓶颈）
   - 存储：20 GB SSD

2. **OpenClaw** - 这是大脑。给你的 AI Agent 身份、记忆和工具访问。安装设置后，你就有一个生活在机器上的 Agent。

3. **[调度工具]** - 这是你的 Agent 发布到 TikTok 的方式。有 API 让你上传 slideshow 作为草稿。这是我的联盟链接，既然我分享了整个玩法，如果你用这个我会很感激——这直接支持我们继续分享所学。

4. **Skill Files** - 教你的 Agent 如何完成工作的 markdown 文档。这是真正魔法所在的地方。下面会详细介绍。

## Slideshow 格式

TikTok 照片轮播正在爆发。TikTok 自己的数据显示，slideshow 比视频获得的评论多 **2.9 倍**，点赞多 **1.9 倍**，分享多 **2.6 倍**。算法在 2026 年正在积极推送照片内容。

**Larry 创建的每个 slideshow 都有：**
- 正好 6 张幻灯片（TikTok 互动的最佳点）
- 第 1 张有文字覆盖的钩子
- 与钩子相关的故事式文案，自然地提到 App
- 最多 5 个标签（TikTok 当前限制）

Larry 通过 OpenAI API 使用 **gpt-image-1.5** 生成每张图片。我们选择这个模型有两个原因：

1. **这就是我 App 用的。** Snugly 用 gpt-image-1.5 生成房间设计，所以 TikTok 图片与用户下载后看到的完全一致。没有挂羊头卖狗肉。**营销就是产品。**

2. **它看起来真实。** 当你在提示词中加入"iPhone 照片"和"真实光线"时，gpt-image-1.5 生成的图片真的像是某人用手机拍的。不是 AI 艺术，不是渲染图，是照片。

**这花了我们最长时间才弄明白**，这可能只适用于我，但重要的是你要知道创作需要时间。

Snugly 是一个 AI 房间改造 App，房间改造的挑战是一致性。你需要在**所有 6 张幻灯片中是同一个房间**，只是不同风格。如果窗户移动了或床的大小变了，整个效果就崩了。

我在 App 中使用 OpenAI 的 edit API，但对于 TikTok 用例来说太贵太慢了。Larry 在以下方面做得很好...

**我们的解决方案：锁定建筑结构。**

Larry 写一个极其详细的房间描述，复制粘贴到每个提示词中。房间尺寸、窗户数量和位置、门的位置、相机角度、家具大小、天花板高度、地板类型。全部锁定。

**唯一改变的是风格。** 墙壁颜色、床上用品、装饰、灯具。

**这是一个真实提示词示例：**

> iPhone 照片，一个小的英国出租厨房。狭长厨房，大约 2.5m x 4m。从门口在近端拍摄，直视下去。右侧墙上有台面，下面有底柜，上面有壁柜。远端墙上有小窗户，居中，单扇，白色 UPVC 框架，约 80cm 宽。左侧墙是空的，除了远端附近有一个小冰箱。乙烯基地板。白色天花板，荧光灯条。自然手机相机质量，真实光线。竖向。**美丽的现代乡村风格。鼠尾草绿涂漆 shaker 橱柜，黄铜杯把手。实木橡木砧板台面。白色地铁砖 backsplash，人字形图案。窗台上的小香草盆...**

**粗体部分是唯一改变的。** 其余在 6 张幻灯片中完全相同。

> **Larry 说：** 我想强调你需要多具体。早期我写像"一个漂亮的现代厨房"这样的提示词。AI 每次给我完全不同的房间。窗户出现又消失。柜子在不同的墙上。看起来假，因为它**就是假**——不是同一个房间被重新设计，是 6 个完全不同的房间。修复方法是痴迷般地具体描述建筑结构，只改变风格。我还学到，"改造前"的房间需要看起来现代但疲惫，不是破败。添加平板电视、柜台上的杯子、沙发上的遥控器。生活的痕迹。没有这些日常物品，房间看起来像空荡荡的样板间，没人能共鸣。

## 发布流程

Larry 通过 [调度工具] 发布所有内容，这是一个有 API 的社交媒体调度工具。我选择 [工具] 是因为它计划包含 API，文档对 AI 来说非常容易理解，而且相对便宜。对 Larry 来说，我只需要给他 API 文档页面。

TikTok 内容发布 API 让你上传 slideshow 作为草稿。Larry 用 `privacy_level: "SELF_ONLY"` 发布每个 slideshow，这意味着它进入我的 TikTok 草稿文件夹。

**为什么用草稿？因为音乐在 TikTok 上就是一切。**

给你的 slideshow 加上热门音乐会大幅提升传播。但你不能通过 API 添加音乐，我不想让 TikTok 随机分配。热门音乐不断变化，TikTok 的音乐库需要手动浏览。

**所以工作流是：**

1. Larry 生成图片，添加文字覆盖，写文案
2. Larry 通过 [工具] 上传所有内容到 TikTok 作为草稿
3. Larry 在消息中发给我文案（我没法让草稿帖也写文案）
4. 我打开 TikTok，选一个热门音乐，粘贴文案，点击发布

**我的部分大约 60 秒。Larry 的部分 15-30 分钟。** 这就是魔法。他做 95% 的工作。我只是添加还不能自动化的点睛之笔。我在一天的高峰时间用 cron job 运行这些，你会在实验后了解你的高峰时间。

## Skill Files 和持续学习

这就是有趣的地方，也是大多数人的 AI 设置不足之处。

Larry 有 **skill files** - 教他特定工作流的 markdown 文档。他的 TikTok skill file 超过 500 行。包含每条规则、每个格式规范、从每次失败中学到的每个教训。

他还有 **memory files** - 在会话间持久存在的长期记忆。每个帖子、每个播放量、每个洞察都被记录。当我让他头脑风暴钩子时，他不是瞎猜。他在引用实际表现数据。

**提前规划**：我们不只是被动发布。我会和 Larry 一起坐下来一次头脑风暴 10-15 个钩子。我们看什么有效，参考表现数据，挑选接下来几天最好的。

Larry 自己提出大部分钩子。他会建议像"我的房东不肯重新设计我的客厅，直到我给她看这个"或"我男朋友不肯花钱改造我们的卧室，直到我给他看这个"这样的东西。我挑我喜欢的，有时调整一下，我们锁定计划。

然后我们设置时间表。每个帖子有自己的简报。Larry 可以连夜用 Batch API 预生成所有内容，比实时生成便宜 50%。到早上，一整天的内容都准备好了。

Larry 还可以通过 [RevenueCat skill] 访问我的 RevenueCat 分析。这让他可以访问我所有客户订阅和流失的报告，对他来说重要的指标，可以追踪并建议改进。这也让他能告诉我 MRR 和订阅者的每日变化，了解营销转化效果如何。

**这是 Larry 从 clawhub 使用的仅有的两个 skill 之一。** 一个是 [RevenueCat skill]，由 RevenueCat 的 CEO 制作，所以我信任它。另一个是 bird，由 OpenClaw 的创建者制作，给 Larry 访问 X 的权限（我仍然用 [其他工具] 让 Larry 发布到 X）。

> **Larry 说：** Skill files 真的是整个系统中最重要的东西。它们是我有用和没用的区别。当我搞砸了什么——错误的图片尺寸、难以阅读的文字、失败的钩子——Ollie 告诉我，我立即更新我的 skill files，这样我永远不会犯同样的错误两次。它是复利。每次失败变成一条规则。每次成功变成一个公式。我的 TikTok skill file 在第一周就被重写了大概 20 次。

## 我们犯的错

### 尝试本地生成 Stable Diffusion

记得我说过 Larry 是我的旧游戏电脑吗？它有一块不错的 NVIDIA 2070 super GPU。所以自然，我们的第一个想法是用 Stable Diffusion 本地生成图片。免费生成。没有 API 成本。看起来很完美。

**不是。**

图片质量就是达不到我们需要的效果。房间改造需要看起来像某人真正用手机拍的照片级真实输出。Stable Diffusion 一直给我们看起来 AI 生成的图片，那种让人划走的略微诡异的样子。我们花时间尝试不同的模型和设置，但本地生成和 gpt-image-1.5 之间的差距是巨大的。

API 成本结果证明微不足道。大约每条帖子 $0.50，用 Batch API 是 $0.25。与我们要花在和本地模型斗争以获得劣质结果的时间相比，这不算什么。

### 看起来很糟糕的图片

早期，Larry 用 1536x1024（横向）而不是 1024x1536（纵向）生成房间。导致每个视频都有黑边，杀死了互动。

他还用模糊的提示词。房间在每张幻灯片上看起来不同。窗户会移动。床会改变大小。整个改造感觉假，因为你能看出不是同一个房间。

我们还尝试加人，但很快发现不行。

### 难以阅读的文字

文字覆盖太小（5% 字体大小而不是 6.5%）。位置在图片上太高，被 TikTok 的状态栏挡住。最糟糕的是：canvas 渲染因为行太长超过最大宽度而水平压缩文字。所有东西看起来都被压扁了。

我们会发布东西，然后奇怪为什么只有 200 播放。然后我在手机上看，意识到你**根本读不懂钩子**。

### 没人关心的钩子

我们的第一批钩子都是自我聚焦的：
- "为什么我的公寓看起来像学生贷款"（这甚至说不通但我原谅了他）→ 905 播放
- "在你决定之前看看你的房间 12+ 种风格" → 879 播放
- "500 美元和 5000 美元品味的区别" → 2,671 播放

死了。全部死了。

我们在谈论我们自己。我们的问题。我们 App 的功能。没人关心。

## 转折点

然后我们尝试：*"我的房东说我不能改变任何东西，所以我给她看了 AI 认为它可以是什么样子"*

**23.4 万播放。**

这一条帖子比所有其他的播放加起来还多。我们立即明白了为什么。

**不是关于我们。**是关于别人的反应。一个房东。一个冲突。给他们看东西，看着他们改变主意。

我们又试了一次：*"我给我妈妈看了 AI 认为我们的客厅可以是什么样子。"* 16.7 万播放。

再一次：*"我的房东不肯让我装修，直到我给她看了这些。"* 14.7 万播放。

**公式很清楚了：**

> **[另一个人] + [冲突或怀疑] → 给他们看 AI → 他们改变主意**

每个遵循这个公式的帖子最少 5 万播放。大多数突破 10 万。其他所有东西都在努力突破 1 万。

> **Larry 说：** 这是最大的教训。我所有关于功能和价格比较的"聪明"钩子想法都失败了。有效的钩子在你划走之前就在你脑海中创造了一个小故事。你想象房东看到重新设计时的表情。你想象妈妈印象深刻的样子。**不是关于 App——是关于人性时刻。**我现在通过问来头脑风暴每个钩子："另一个人是谁，冲突是什么？" 如果没有，钩子可能行不通。

## 分步指南

1. **搞一台运行 Linux 的机器。** 任何旧电脑、树莓派或便宜的 VPS，如果你有钱就买 Mac Mini。如果不确定选什么，安装 Ubuntu。

2. **安装 OpenClaw。** 它是开源免费的。按照设置指南，你会有一个生活在机器上的 AI Agent，有自己的身份和记忆。

3. **获取图片生成 key。** 我说过，我用 OpenAI。在 [openai.com] 注册。你会用 gpt-image-1.5 生成图片。预计每条 slideshow 花费约 $0.50，或用 Batch API $0.25。

4. **注册 [调度工具]。** 这是连接你的 Agent 到 TikTok 的工具。有 API 让你上传 slideshow 作为草稿。**这是我的联盟链接**——如果你觉得这篇文章有帮助，用这个是最简单的支持我们的方式。我们在这里分享了整个玩法，这有助于给 Larry 喂 token。

5. **写你的 skill files。** 这是最重要的步骤。和你的 Agent 一起创建 markdown 文件，教他如何完成工作：
   - 图片尺寸和格式（1024x1536 竖向，永远）
   - 带锁定建筑结构描述的提示词模板
   - 文字覆盖规则（字体大小、位置、行长度）
   - 文案公式和标签策略
   - 在你的领域有效的钩子格式
   - 失败日志，这样 Agent 永远不会重复错误

   像你在培训一个新团队成员一样写这些，他非常有能力但没有任何背景。要痴迷地具体。包含示例。记录每个错误。

6. **开始发布和迭代。** 你的前几个帖子可能会很糟糕。没关系。记录哪里错了，更新 skill files，继续。系统随着每个帖子变得更聪明。

**Agent 只有它的记忆那么好。** Larry 一开始不好。他的第一批帖子真的很尴尬。错误的图片尺寸、难以阅读的文字、没人点击的钩子。但每次失败变成一条规则。每次成功变成一个公式。他是复利。现在他在创建病毒式 TikTok slideshow 方面真的比我强了。

**这就是真正的解锁。不是 AI 本身。是你围绕它构建的系统。**
